    Hermes Agent 完全教程

    什么是 Hermes Agent？

    Hermes Agent 是由 Nous Research 开发的开源 AI 代理框架，可连接多种大语言模型（LLM），通过丰富的工具链（终端、文件操作、网络搜索、代码执行等）自动完成复杂任务。它支持 QQ、Telegram、Discord 等多平台接入，可作为你的私人 AI 助手全天候在线。

    核心特性

    • 多模型支持 — 可接入 OpenAI、Anthropic、DeepSeek、Google Gemini、本地 Ollama 等数十种 LLM
    • 多平台接入 — QQ、Telegram、Discord、微信、Slack、Matrix 等
    • 工具链丰富 — 终端命令、文件读写、代码执行、网页搜索、浏览器控制、GitHub 集成
    • 技能系统 — 可复用的工作流模板，支持自定义技能
    • 子代理（Delegation） — 将子任务分配给本地模型执行，节省 API 额度
    • 辅助模型 — Vision（看图）、Compression（压缩上下文）、Session Search（历史检索）等专用通道
    • 持久记忆 — 跨会话记住用户偏好和环境信息
    • 定时任务（Cron） — 按计划自动执行任务并推送结果
    • Fallback 降级 — API 不可用时自动切换到备用模型

    一、安装

    环境要求

    项目 最低要求 推荐
    操作系统 Ubuntu 20.04+ / macOS Ubuntu 24.04
    Python 3.10+ 3.12
    RAM 2GB 4GB+
    磁盘空间 5GB 20GB
    Node.js 18+ 20+

    安装步骤

    1. 安装系统依赖

    sudo apt update && sudo apt install -y python3 python3-pip python3-venv git curl
curl -fsSL https://deb.nodesource.com/setup_20.x | sudo -E bash -
sudo apt install -y nodejs

    2. 安装 Hermes Agent

    # 全局安装（推荐）
pip install hermes-agent

# 验证安装
hermes --version

    3. 初始化

    hermes init

    此命令会创建 ~/.hermes/ 目录，包含 config.yaml（主配置）和 .env（敏感信息）。

    4. 安装 Ollama（可选，用于本地推理）

    curl -fsSL https://ollama.com/install.sh | sh
ollama pull qwen2.5:3b-instruct-q4_K_M
ollama pull moondream  # 轻量看图模型

    二、配置

    配置文件结构

    Hermes Agent 使用两个配置文件：

    文件 作用 位置
    config.yaml 所有配置项（模型、通道、工具、安全等） ~/.hermes/config.yaml
    .env 敏感信息（API Key、密码、Token） ~/.hermes/.env

    主模型配置

    model:
  provider: deepseek          # 主模型提供商
  model: deepseek-v4-flash    # 模型名
  api_key: ''                 # 可在 .env 中设置 DEEPSEEK_API_KEY
  base_url: https://api.deepseek.com
  temperature: 0.1
  context_length: 65536

    支持的提供商

    提供商 provider 值 需配置的环境变量
    OpenAI openai OPENAI_API_KEY
    Anthropic anthropic ANTHROPIC_API_KEY
    DeepSeek deepseek DEEPSEEK_API_KEY
    Google Gemini google GOOGLE_API_KEY
    OpenRouter openrouter OPENROUTER_API_KEY
    本地 Ollama custom 在 config 中设 base_url
    AWS Bedrock bedrock AWS IAM 凭证
    MiniMax minimax MINIMAX_API_KEY

    多模型架构（API 优化方案）

    将付费 API 用于核心推理，本地模型处理辅助任务，大幅节省 API 费用：

    # ── 主模型：付费 API ──
model:
  provider: deepseek
  model: deepseek-v4-flash

# ── 辅助模型：全部走本地 ──
auxiliary:
  vision:
    provider: custom
    model: moondream                # 看图（轻量多模态）
    base_url: http://nas:11434/v1
    timeout: 60

  compression:
    provider: custom
    model: qwen2.5:3b-instruct      # 上下文压缩
    base_url: http://nas:11434/v1

  web_extract:
    provider: custom
    model: qwen2.5:3b-instruct      # 网页提取
    base_url: http://nas:11434/v1

  session_search:
    provider: custom
    model: qwen2.5:3b-instruct      # 历史检索
    base_url: http://127.0.0.1:11434/v1

  title_generation:
    provider: custom
    model: qwen2.5:3b-instruct
    base_url: http://127.0.0.1:11434/v1

# ── 子代理：本地模型执行子任务 ──
delegation:
  model: gemma4:e2b
  provider: custom
  base_url: http://nas:11434/v1
  max_concurrent_children: 2
  child_timeout_seconds: 600

# ── Fallback：API 挂了自动降级 ──
fallback_model:
  provider: custom
  model: gemma4:e2b
  base_url: http://nas:11434/v1

    优化效果：API token 消耗减少 60-70%，所有辅助操作（看图、检索、压缩、分类、子任务）零 API 成本。

    三、启动与运行

    启动 CLI 对话

    hermes run

    启动 Gateway 服务（生产模式）

    Gateway 是 Hermes Agent 的长期运行服务，支持多平台消息接入。

    hermes gateway

    默认监听 http://0.0.0.0:8001。

    配置 systemd 开机自启

    # /etc/systemd/system/hermes-gateway.service
[Unit]
Description=Hermes Gateway
After=network.target

[Service]
Type=simple
User=harry
WorkingDirectory=/home/harry
ExecStart=/home/harry/.local/bin/hermes gateway
Restart=always
RestartSec=5

[Install]
WantedBy=multi-user.target

    sudo systemctl daemon-reload
sudo systemctl enable --now hermes-gateway

    四、消息平台接入

    QQ 接入

    1. 在 QQ 开放平台 创建机器人，获取 APP_ID 和 CLIENT_SECRET
    2. 配置 .env：
    QQ_APP_ID=your_app_id
QQ_CLIENT_SECRET=your_client_secret
QQ_ALLOWED_USERS=32F844475EE98D76F821B9A94E5C14AF
QQBOT_HOME_CHANNEL=32F844475EE98D76F821B9A94E5C14AF
    1. 在 config.yaml 设置 QQ 回调地址指向 Gateway

    Telegram 接入

    telegram:
  allowed_chats: '-1001234567890'
  reactions: true
  channel_prompts: {}

    需在 .env 中配置 TELEGRAM_BOT_TOKEN。

    Discord 接入

    discord:
  require_mention: true
  auto_thread: true
  reactions: true

    需在 .env 中配置 Discord Bot Token。

    五、核心功能

    1. 工具集（Tools）

    Hermes 可调用以下工具完成任务：

    工具 功能 适用场景
    terminal 执行 shell 命令 部署、运维、数据处理
    read_file / write_file 读写文件 代码编辑、文档生成
    patch 文件查找替换 代码修改
    search_files 文件内容/名称搜索 代码库导航
    web_search 网页搜索 信息获取
    vision_analyze 图片分析 看图识别
    execute_code 运行 Python 数据处理、自动化
    delegate_task 派生子代理 并行子任务
    cronjob 定时任务 定期报告、监控
    memory 持久记忆 记住用户偏好
    send_message 多平台发送 推送通知

    2. 技能系统（Skills）

    技能是可复用的工作流模板，存储在 ~/.hermes/skills/。

    # 查看可用技能
hermes skills list

# 加载技能
hermes skill load <name>

    技能示例：代码审查、架构图绘制、模型评测、系统运维等。

    3. 子代理（Delegation）

    将复杂任务拆分给子代理并行执行，每个子代理有独立会话和工具集。

    delegation:
  model: gemma4:e2b           # 子代理使用的模型
  max_concurrent_children: 3  # 最大并行数
  max_spawn_depth: 2          # 嵌套深度
  orchestrator_enabled: true  # 允许子代理再派生子代理

    4. 定时任务（Cron）

    # 创建每日报告
hermes cron create \
  --name "每日简报" \
  --schedule "0 9 * * *" \
  --prompt "生成今天的系统状态简报"

# 查看任务列表
hermes cron list

# 删除任务
hermes cron remove <job_id>

    5. 持久记忆

    Hermes 会自动记住用户信息，跨会话保持上下文。

    memory:
  memory_enabled: true        # 启用记忆
  user_profile_enabled: true  # 启用用户画像
  memory_char_limit: 2200     # 记忆字符上限

    六、API 优化架构（实战案例）

    以易弘机构的部署为例：

                     ┌──────────────────────┐
  用户 ──→       │  DeepSeek V4 Flash    │ ← 仅复杂推理（付费 API）
                 │     （主力大脑）       │
                 └──────────┬───────────┘
                            │
         ┌──────────────────┼──────────────────┐
         ▼                  ▼                  ▼
  ┌──────────────┐  ┌──────────────┐  ┌──────────────┐
  │  子代理任务    │  │  辅助处理     │  │  工具执行     │
  │  NAS          │  │  NAS/本地     │  │  本地终端     │
  │  gemma4:e2b   │  │  moondream   │  │  文件/代码    │
  │               │  │  qwen2.5:3b  │  │              │
  └──────────────┘  └──────────────┘  └──────────────┘
     零 API            零 API            零 API

    分层策略：

    层级 模型 位置 成本
    🧠 核心推理 DeepSeek V4 Flash API 💰 付费
    🧑‍💼 子代理 gemma4:e2b NAS 免费
    👁️ 看图 moondream NAS 免费
    📝 压缩/分类 qwen2.5:3b NAS 免费
    🔍 检索/标题 qwen2.5:3b 本机 免费
    🛡️ Fallback gemma4:e2b NAS 免费

    七、最佳实践

    配置管理

    # 修改配置前备份
cp ~/.hermes/config.yaml ~/.hermes/config.yaml.bak

# 恢复配置
cp ~/.hermes/config.yaml.bak ~/.hermes/config.yaml
# 重启 Gateway

    安全建议

    1. API Key 只放在 .env 中，不要提交到 git
    2. 开启 TIRITH 安全扫描（默认开启）
    3. 限制允许的用户 ID：QQ_ALLOWED_USERS、telegram.allowed_chats
    4. 敏感操作需审批：approvals.mode: manual

    模型选择建议

    服务器 RAM 推荐模型 用途
    NAS（8GB） 8GB gemma4:e2b（6.8GB）+ moondream（1.6GB） 子代理 + 看图
    低配服务器（4GB） 4GB qwen2.5:3b（1.8GB） 文字辅助
    MacBook M2 Max 64GB 任意大模型 移动工作站
    Windows 工作站 32GB+ deepseek-r1:14b+ 本地推理

    八、故障排查

    Gateway 启动失败

    # 查看日志
journalctl -u hermes-gateway -f

# 检查配置文件语法
hermes config validate

    Ollama 连接失败

    # 确认 Ollama 运行
curl http://nas:11434/api/tags

# 确认监听 0.0.0.0
ss -tlnp | grep 11434

    看图超时

    • 检查模型是否支持多模态（如 moondream、gemma3:4b、llava）
    • 首次加载需要时间，设置合理的 timeout
    • NAS 内存不足时，换用更小的模型

    子代理不工作

    # 检查 delegation 配置是否正确
hermes config get delegation

# 确认子代理模型可访问
curl http://nas:11434/api/generate -d '{"model":"gemma4:e2b","prompt":"hi"}'
