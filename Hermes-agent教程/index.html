    Hermes-agent 安装教程

    简介

    Hermes-agent 是一个基于 LLM 的智能代理框架，部署在 Ubuntu 服务器上，可通过 API 调用实现任务自动化执行。本文档记录其在 Ub-server (192.168.31.84) 上的完整安装步骤。

    环境要求

    项目 要求
    操作系统 Ubuntu 20.04+
    Python 3.10+
    RAM ≥ 8GB
    磁盘空间 ≥ 20GB
    网络 可访问 Ollama (192.168.31.187:11434)

    步骤一：安装依赖

    # 更新系统
sudo apt update && sudo apt upgrade -y

# 安装 Python 3.10+ 及 pip
sudo apt install -y python3 python3-pip python3-venv git curl

# 安装 Node.js
curl -fsSL https://deb.nodesource.com/setup_20.x | sudo -E bash -
sudo apt install -y nodejs

# 验证版本
python3 --version
node --version

    步骤二：安装 Ollama（可选，若使用远程 Ollama 可跳过）

    Hermes-agent 依赖 Ollama 提供 LLM 推理能力，推荐使用飞牛 NAS 上的远程 Ollama，或本地安装。

    # 本地安装 Ollama
curl -fsSL https://ollama.com/install.sh | sh

# 启动 Ollama（监听所有网卡，供远程调用）
OLLAMA_HOST=0.0.0.0 ollama serve &

# 拉取推荐模型
ollama pull llama3.1:8b-instruct-q4_K_M

    步骤三：获取 Hermes-agent 代码

    # 创建部署目录
sudo mkdir -p /opt/hermes-agent
sudo chown $USER:$USER /opt/hermes-agent
cd /opt/hermes-agent

# 克隆代码（请替换为实际仓库地址）
git clone https://github.com/your-org/hermes-agent.git .

# 切换到稳定分支
git checkout main

    步骤四：配置环境变量

    cp .env.example .env
nano .env

    .env 关键配置项：

    OLLAMA_BASE_URL=http://192.168.31.187:11434
OLLAMA_MODEL=llama3.1:8b-instruct-q4_K_M
HOST=0.0.0.0
PORT=8000
API_KEY=your-secret-api-key
LOG_LEVEL=info

    步骤五：安装 Python 依赖

    cd /opt/hermes-agent
python3 -m venv venv
source venv/bin/activate
pip install -r requirements.txt

    步骤六：运行 Hermes-agent

    开发模式

    cd /opt/hermes-agent
source venv/bin/activate
uvicorn main:app --host 0.0.0.0 --port 8000 --reload

    生产模式（systemd）

    sudo nano /etc/systemd/system/hermes-agent.service

    [Unit]
Description=Hermes Agent Service
After=network.target

[Service]
Type=simple
User=harry
WorkingDirectory=/opt/hermes-agent
Environment="PATH=/opt/hermes-agent/venv/bin"
ExecStart=/opt/hermes-agent/venv/bin/python /opt/hermes-agent/main.py
Restart=always
RestartSec=5

[Install]
WantedBy=multi-user.target

    sudo systemctl daemon-reload
sudo systemctl enable hermes-agent
sudo systemctl start hermes-agent
sudo systemctl status hermes-agent

    步骤七：验证安装

    curl http://localhost:8000/health

curl -X POST http://localhost:8000/chat \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer your-secret-api-key" \
  -d '{"message": "你好，请介绍一下你自己"}'

    常见问题

    Ollama 连接失败

    # 在 NAS 上执行，确认 Ollama 监听 0.0.0.0
sudo OLLAMA_HOST=0.0.0.0 ollama serve &

# 测试连通性
curl http://192.168.31.187:11434/api/tags

    端口被占用

    sudo lsof -i :8000
# 修改 .env 中的 PORT

    项目 值
    部署服务器 Ub-server (192.168.31.84 / 10.166.37.64)
    Ollama 地址 飞牛 NAS (192.168.31.187:11434)
    服务端口 8000
    配置文件 /opt/hermes-agent/.env
